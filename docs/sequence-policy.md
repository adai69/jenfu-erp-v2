# 序號模組強制規則

鉦富機械 ERP V2 規定：任何新建或調整的模組不得自行產生編碼，所有流水號必須透過序號模組進行統一治理。以下為具體規範：

## 1. 規則說明
- **唯一來源**：`SequenceDirectory` 所管理的設定為全系統唯一參考，不得於模組本身另行定義 prefix、padding 或遞增規則。
- **串接義務**：所有 CRUD 流程在「建立」資料前必須先呼叫序號服務取得正式編碼；若序號服務回傳失敗，整筆流程需中止。
- **變更治理**：調整 prefix、padding、nextNumber、scope 等設定，需透過序號模組內的「調整編碼」動作並保留操作紀錄，禁止於其他模組直接覆寫。
- **軌跡留存**：序號服務需記錄取號、預覽、鎖定等操作，以符合鉦富的財務與品質稽核需求。

## 2. 實作建議
1. 引用 `src/lib/sequenceManager.ts` 提供的 API（`peekSequence`、`issueSequence` 等）作為唯一管道。
2. 未來連接資料庫時，序號模組負責交易鎖與回寫下一號；前端僅負責顯示結果。
3. 模組文件需註明「序號來源：Sequences」，並於程式碼層面建立測試或型別限制，阻擋自行輸入任意字串。

## 3. 例外流程
- 暫時允許測試環境 Mock，但需在 PR/文件標註「僅供測試」，不得進入正式部署。
- 若外部系統需導入歷史資料，需先透過序號模組提供的「批次同步」功能補寫 nextNumber，再進行匯入。
